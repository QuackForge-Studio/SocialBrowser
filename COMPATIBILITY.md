## Electron 42 Fallback

If Electron 43 introduces breaking changes or ABI incompatibilities, the following fallback versions should be used:

| Package | Version | Notes |
|---|---|---|
| electron | 42.7.0 | Fallback. Chromium 148, V8 14.8, Node.js v24.15.0 |
| better-sqlite3 | 12.11.1 | Same version — compatible with Electron 42 Node.js v24.15.0 |
| sqlite-vec | 0.1.9 | Same version — pure SQLite extension, no Electron dependency |
| @electron/rebuild | 3.6.3 | Same version |
| electron-builder | 26.15.3 | Same version |

## Switching to Fallback

To switch from Electron 43 to Electron 42:

1. Update `package.json`:
   - Change `"electron": "43.1.1"` to `"electron": "42.7.0"`
2. Delete `pnpm-lock.yaml`
3. Run `pnpm install`
4. Run `pnpm rebuild` (compiles native addons for Electron 42 ABI)
5. Verify with `node scripts/verify-addons.js`

## Native Addon Rebuild

Native addons (better-sqlite3, sqlite-vec) must be rebuilt for each Electron version:

```bash
# After pnpm install (automatic via postinstall)
pnpm install

# Manual rebuild if needed
pnpm rebuild

# Verify
node scripts/verify-addons.js
```

## Lockfile

- Lockfile: `pnpm-lock.yaml`
- Generated by `pnpm install` with pinned exact versions
- Must be committed to the repository
- Use `pnpm install --frozen-lockfile` in CI to ensure reproducible builds
